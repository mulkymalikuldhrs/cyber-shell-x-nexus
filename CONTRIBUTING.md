# Contributing to CyberShellX Nexus

First off, thank you for considering contributing to CyberShellX Nexus! It's people like you that make CyberShellX Nexus such a great tool.

## Where do I go from here?

If you've noticed a bug or have a question, [search the issue tracker](https://github.com/mulkymalikuldhrs/cyber-shell-x-nexus/issues) to see if someone else has already reported the issue. If not, feel free to create a new issue.

## Fork & create a branch

If you're going to fix a bug or implement a new feature, you'll need to create a fork of the repository and create a new branch.

1.  Fork the repository.
2.  Create a new branch from `main`.

## Commit your changes

- Make sure your commit messages are descriptive.

## Submitting a pull request

When you're ready to submit a pull request, make sure you do the following:

1.  Run the test suite and make sure all tests pass.
2.  Include a description of the changes you've made.
3.  Reference any issues that your pull request addresses.

Thank you for your contribution!
